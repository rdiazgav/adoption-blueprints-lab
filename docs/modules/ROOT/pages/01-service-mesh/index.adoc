= Workshop 01 — Service Mesh with OpenShift Service Mesh 3.x
:navtitle: Overview

This workshop installs OpenShift Service Mesh 3.x (Sail Operator) on top of the RetailFlow application and walks through five progressive scenarios demonstrating the core value of a service mesh: visibility, secure ingress, traffic control, failure isolation, and zero-trust authorization.

Estimated total time: *60–90 minutes*

== What you will learn

* How to install OpenShift Service Mesh 3.x using the Sail Operator
* How sidecar injection adds Envoy proxies to every pod without code changes
* How mTLS encrypts all service-to-service traffic automatically
* How to configure a Gateway API ingress that flows through the mesh
* How to do a canary release with weighted traffic splitting
* How a circuit breaker isolates failures in a downstream service
* How AuthorizationPolicy enforces zero-trust between services using SPIFFE identities

== Prerequisites

* xref:../00-setup/index.adoc[Workshop 00] completed — RetailFlow deployed and all pods healthy in `retailflow`
* `cluster-admin` permissions on OpenShift 4.14+ or CRC (>= 16 GB RAM)
* `oc` CLI configured and logged in

Verify the baseline before starting:

[source,bash]
----
oc get pods -n retailflow --field-selector=status.phase=Running | wc -l
# Expected: 11 or more running pods
----

== Quick Start — Apply Everything at Once

To deploy all mesh resources in a single command:

[tabs]
====
CRC::
+
--
[source,bash]
----
oc apply -k workshops/01-service-mesh/deploy/overlays/crc
----
--
External cluster::
+
--
[source,bash]
----
oc apply -k workshops/01-service-mesh/deploy/overlays/demo-platform
----
--
====

Then follow the verification steps in each scenario to understand what was deployed.
