= Workshop 03 — OpenShift Pipelines
:navtitle: OpenShift Pipelines

[.lead]
Replace the manual `oc apply -k` workflow with automated CI pipelines that build, test, and deploy every RetailFlow service on every commit — using OpenShift Pipelines (Tekton) as the engine.

Estimated total time: *90–120 minutes* | Difficulty: *Intermediate*

== What You Will Learn

* How Tekton models CI/CD as Kubernetes-native resources: Task, Pipeline, PipelineRun, Workspace
* How to build and push a Quarkus service image using Buildah inside a Pipeline
* How to run Maven tests as a quality gate before the image is built
* How to build Python and Node.js services with a language-agnostic `generic-service-pipeline`
* How to connect a GitHub webhook to a Tekton EventListener so every `git push` launches a PipelineRun automatically

== The Problem It Solves

Until Workshop 03, deploying RetailFlow meant running `oc apply -k deploy/overlays/crc` by hand.
That works in a workshop, but in a real team, manual deploys are slow, error-prone, and invisible:
a developer commits a fix, hands off to operations, waits for a Slack acknowledgment, and hopes the correct overlay was used.

OpenShift Pipelines closes this gap. Every push triggers a PipelineRun that:

. Clones the source from GitHub
. Runs Maven tests (for Quarkus services)
. Builds the container image with Buildah (rootless, OCI-compliant)
. Pushes the image to Quay
. Updates the Deployment in the `retailflow` namespace and waits for rollout

All of this happens automatically, is fully auditable as Kubernetes objects, and requires zero changes to the application code.

== Architecture

[source]
----
Developer
    │
    │  git push
    ▼
GitHub repository
    │
    │  HTTP POST (webhook)
    ▼
OpenShift Route  ──► EventListener pod
                          │
                    Interceptors
                    (validate HMAC signature + filter branch)
                          │
                    TriggerBinding
                    (extract git-url, git-revision, service-name)
                          │
                    TriggerTemplate
                          │
                          ▼
                    PipelineRun created
                          │
          ┌───────────────┼───────────────────┐
          ▼               ▼                   ▼
     Task: git-clone  Task: maven-build  Task: buildah
     (Tekton Hub)     (custom)           (Tekton Hub)
          │               │                   │
          └───────────────┴───────────────────┘
                          │
                          ▼
                    Quay registry
                    (quay.io/rdiazgav/retailflow-<service>:<sha>)
                          │
                          ▼
                    Task: deploy-to-openshift
                    (oc set image + oc rollout status)
                          │
                          ▼
                    OpenShift Deployment updated
----

== Quick Start

=== Step 1 — Install and configure everything

[source,bash]
----
cd workshops/03-pipelines
bash scripts/setup.sh
----

After `setup.sh` completes, create and apply your Quay registry credentials:

[source,bash]
----
cp deploy/base/01-rbac/quay-credentials.yaml.template quay-credentials.yaml
# Edit quay-credentials.yaml and paste your base64-encoded docker config
oc apply -f quay-credentials.yaml -n retailflow
oc secret link pipeline-sa quay-credentials --for=pull,mount -n retailflow
----

=== Step 2 — Run the catalog pipeline

[source,bash]
----
bash scripts/run-pipeline.sh catalog
----

Follow logs in real time. When the pipeline completes, verify the deployment was updated:

[source,bash]
----
oc get deployment catalog -n retailflow \
  -o jsonpath='{.spec.template.spec.containers[0].image}'
----
